# Shadowsocks-Rust 负载均衡机制详解

在 `shadowsocks-rust` 中，`sslocal` 能够通过配置多个服务器实现负载均衡。其核心逻辑在于通过动态探测服务器的延迟（RTT）与可用性，结合用户定义的权重，自动选择最优节点。

## 1. 核心配置参数解析

负载均衡的行为主要受配置文件中 `servers` 列表及 `balancer` 模块的影响。

### 1.1 服务器定义 (`servers`)
当 `servers` 数组中包含多个有效的服务器配置时，负载均衡功能自动启用。

*   **`tcp_weight` / `udp_weight`**: 
    *   **取值**: `0.0` 到 `1.0`。**默认值: `1.0`**。
    *   **影响**: 权重越高，该服务器在计算得分时越占优势。在 RTT（延迟）相近的情况下，权重高的服务器会被优先选中。

### 1.2 均衡器控制 (`balancer`)
该模块定义了如何评估服务器状态：

*   **`max_server_rtt`**: 
    *   判定服务器“可用”的最大延迟阈值（单位：秒）。**默认值: `5`**。
    *   如果某个服务器的探测延迟超过此值，均衡器会将其视为不可用或质量极差，从而降低其选择优先级。
*   **`check_interval`**: 
    *   常规检查周期（单位：秒）。**默认值: `10`**。负载均衡器会按照此间隔对所有非禁用状态的服务器进行 RTT 探测。
*   **`check_best_interval`**: 
    *   针对“当前最优”服务器的高频检查间隔（单位：秒）。**默认值: `5`**。
    *   **作用**: 为了确保选中的服务器始终保持高质量，系统会以更短的周期监控它。一旦发现其延迟抖动，能更快速地切换到备用服务器。

## 2. 负载均衡工作流程

`sslocal` 的负载均衡遵循 **“探测 -> 评估 -> 决策”** 的循环过程。

### 第一阶段：动态探测 (Probing)
负载均衡器在后台启动协程，定期向 `servers` 列表中的节点发送探测包。
*   通过测量建立连接或发送数据包的往返时间获取实时 RTT。
*   记录成功率，若连续探测失败，则标记该节点为失效。

### 第二阶段：评估与调权 (Scoring)
系统并非简单地比较 RTT，而是结合权重计算一个“虚拟得分”：
$$ \text{Score} = \frac{\text{RTT}}{\text{Weight}} $$
*   即使 A 服务器延迟略高于 B，如果 A 的权重显著高于 B，A 仍可能被判定为更优。

### 第三阶段：决策选择 (Selection)
`sslocal` 采用 **最低延迟策略 (Lowest Latency Strategy)**：
1.  **过滤**: 剔除超过 `max_server_rtt` 或完全不可达的服务器。
2.  **排序**: 根据计算出的得分对可用服务器进行升序排列。
3.  **锁定**: 所有的出站流量将优先导向得分最低（即最优）的服务器。

### 第四阶段：失效转移 (Failover)
当主用服务器突然宕机时：
*   当前连接会触发超时或重置。
*   均衡器在下一次请求或高频探测 (`check_best_interval`) 发现故障后，立即从已排序的备选列表中挑选次优节点接管流量。

## 3. 典型配置示例

以下配置展示了两个具有不同优先级的服务器，并自定义了检查频率：

```json
{
    "locals": [{ "local_port": 1080 }],
    "servers": [
        {
            "server": "fast.server.com",
            "server_port": 8388,
            "password": "pass",
            "method": "aes-256-gcm",
            "tcp_weight": 1.0
        },
        {
            "server": "backup.server.com",
            "server_port": 8388,
            "password": "pass",
            "method": "aes-256-gcm",
            "tcp_weight": 0.5
        }
    ],
    "balancer": {
        "max_server_rtt": 3,
        "check_interval": 30,
        "check_best_interval": 10
    }
}
```
